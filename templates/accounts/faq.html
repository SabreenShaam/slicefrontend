General Instructions

Restart the app.

When something does not seem right, the first thing we recommend a user should to do is to log out from the app and log back in again. This often solves a number of issues.

Most Common Enquires

The user cannot log into the app.

Potential issues:
  1. The user may have used the wrong e-mail. The correct e-mail to log in to the app is the same email used to login to Mindbody.
  2. The user may have forgotten their password. The password cannot be reset via Fitopia (as Fitopia simply verifies the credentials in Mindbody). The password will need to be reset by the studio or the user can request a password reset via Mindbody.
  3. The user may have selected the wrong Home Studio. The correct Home Studio is the studio that the user has an account with.

A user may have more than one account at a studio so it’s important to identify which account they wish to login to and use these login credentials.

A user may also have accounts at more than one studio so it’s important that the login credentials, particularly the Home Studio, matches the account they wish to access.

The user does not see external classes on the Find screen or credits in their Wallet

  1. Please encourage the user to log out and log in again.
  2. If the issue persists, the user may have more than one account in your MBO system. You will need to merge these accounts and then ask the user to log out of the app and log in again (using the correct Mindbody login credentials).

A user may have more than one account at a studio so it’s important to identify which account they wish to login to and use these login credentials.

A user may also have accounts at more than one studio so it’s important that the login credentials, particularly the Home Studio, matches the account they wish to access.

The user does not see the correct number of credits in their Wallet

  1. Please encourage the user to log out and log in again.
  2. Please note that the Wallet shows the remaining credits not the ones that have already been used e.g.if the user has 5 credits per month and has used 3 then the remaining 2 will appear in their Wallet.

The user cannot book classes and there are still credits in their Wallet

  1. Please encourage the user to log out and log in again.
  2. If the issue persists, this could be because the user has used their maximum visits per studio ‘Max Per Studio’ for a particular studio and is trying to book into this studio again. You can check this setting in your Studio Dashboard under Manage/External Credits/Max Per Studio, then check the Max Per Studio beside the applicable Pricing Option.
  3. Please also check the start date of their package as credits will show in their wallet prior to the start of their package.

In the monthly report a user appears to have used more credits than their External Credit allocation.

This is likely to be due to timing. The monthly reports are based on a calendar month whereas a users package can begin at any time during the month and is therefore a rolling month. For example, a user may start their package on the 15th January meaning their monthly cycle is 15th Jan to 14th Feb, then 15th Feb to 14th March. If the user uses all of their credits in the second half of their January cycle (1st-14th Feb) and all of their credits in the first half of their February cycle (15th-28 Feb) two months of credits will fall into calendar month February.
If you’re still unable to resolve the issue using the above, we’re happy to help.
